# In 100 Years — 100年後リサーチ

> **目的**：100年後（約2125年）の「常識」と世界を想像し、**そのとき必要とされるのに、いま正しく評価されていないもの**を世界中の情報から見つけ出す。
> 早すぎる確信に「所有」を置くことで、長期の非対称な果実（=ビリオネア級のリターン）を狙うためのリサーチ基盤。

---

## このリポジトリの考え方

おおきなリターンは、**「ほぼ確実に来る未来」**が**「いまは見過ごされている／安い／低ステータス」**な地点に、**「自分が防御できる所有」**を重ねたときに生まれる。

```
期待リターン  ≈  不可避性  ×  評価ギャップ  ×  捕捉可能性
                (来るか)     (安いか)        (自分が握れるか)
```

このリポジトリは、その3つを満たすテーマを系統的に探し、記録し、スコアリングし、確信を育てるための「外部脳」です。

## 使い方（運用ループ）

```
   ① 集める          ② つなぐ            ③ 採点する         ④ 布石を打つ
  シグナル収集  →   トレンド/ドメイン  →  アイデア化＆     →  所有を置く
  (signals)        として束ねる          スコアリング         (投資・参入・
                   (trends/domains)      (ideas + 評価表)      技術・人脈)
        ↑                                                          │
        └──────────────────────── 反証・更新 ←──────────────────────┘
```

1. **集める** — 世界中の論文・特許・統計・規制・辺境のニュースから「弱いシグナル」を拾い、`research/signals/INDEX.md` に記録（テンプレ：`templates/signal.md`）。
2. **つなぐ** — シグナルを `docs/02_domain-map.md` のドメインに分類し、必要なら `research/domains/` で深掘り。
3. **採点する** — 有望テーマを `research/ideas/` にアイデアとして起こし、`docs/01_evaluation-framework.md` の7軸で採点。`INDEX.md` のスコアボードに反映。
4. **布石を打つ** — 高スコア（S/Aティア）のテーマに、捕捉可能な「所有」を置く（株・未公開・土地・特許・規格・人脈・自分の時間）。
5. **反証・更新** — 定期的に「自分が間違っているとしたら？」を問い直し、スコアと確信ティアを更新。

## 原則（迷ったら戻る場所）

- **早すぎることは、間違っていることと区別がつかない。** 価値はコンセンサスより前の確信に宿る。
- **流行（10年）と必然（100年）を混同しない。** 直線外挿を疑う。
- **正しいだけでは儲からない。** 「自分が所有できるボトルネック」まで降りる。
- **退屈・タブー・低ステータスを見くだすな。** 評価ギャップはそこに眠る。
- **反証を歓迎する。** 確信は更新され続けるからこそ価値になる。
